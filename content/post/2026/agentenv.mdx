私は個人でChatGPT PlusとClaude Maxを契約しているが、会社でもCoding Agentの契約がある。
当然、個人アカウントで業務をしてはいけないし、逆も然り。

ブラウザではプロファイル設定で簡単に切り替えられるもののローカルで動くCoding agentはそうもいかない。業務開始時に毎回ログインし直せばいいが、切り替えを忘れると事故になるしそもそも毎回のログインが面倒くさい。業務用に切り替えようと思ったら個人用のブラウザで認証画面が開いて……、面倒。

これを解決する良い手段がある。Claude Codeは`CLAUDE_CONFIG_DIR`、Codexは`CODEX_HOME`をそれぞれ認証情報とかのベースディレクトリとして使用する。つまり、この環境変数を一発で切り替え、別でシェルを開いても現在使っている`CLAUDE_CONFIG_DIR`がそのまま使われるようにすれば良い。そうすれば朝に一発切り替え、退勤後に切り替えるだけで話が済む。

## 実装

元々はシェルスクリプトとして実装していたが、色々機能追加したくなったのでRustで書き直した。

https://github.com/namachan10777/agentenv

```title="agentenv --help"
# agentenv --help                                                                                      ☸ yh1-user (n2-system) 󰂄 36%
Switch Claude Code / Codex profiles (CLAUDE_CONFIG_DIR / CODEX_HOME) per shell. switch, load and remove print eval-able code to stdout; set up the wrapper that evals it with agentenv hook --shell <your-shell>

Usage: agentenv [OPTIONS] [COMMAND]

Commands:
  switch      Switch to (and create if missing) an environment; picks with skim when no name is given
  remove      Remove an environment (the default environment is protected)
  list        List environments; the current one is marked with '*'
  load        Emit eval-able code for the env selected by .agentenv, $AGENTENV_OVERRIDE or the saved state
  hook        Print the shell hook (wrapper function + auto-load on cd)
  completion  Print shell completions
  prompt      Print a prompt segment for the current environment
  starship    Print an example starship configuration
  help        Print this message or the help of the given subcommand(s)

Options:
      --shell <SHELL>  Shell the emitted code / hook / completion targets [default: posix] [possible values: posix, bash, zsh, fish]
  -h, --help           Print help
  -V, --version        Print version
```

hookで`agentenv`をラップして、switchやloadなどを打った際に`agentenv`が出力するシェルスクリプトを`source`することで実装している。
プロジェクトローカルの`.agentenv`ファイルもしくは`$XDG_CONFIG_HOME/agentenv/config.toml`でどの環境を使うかを設定出来るようにしてある。
常に`.agentenv`を置けるようなリポジトリばかりではないため……。

使い方としては、`agentenv hook --shell fish`を打ってその出力をシェルの設定ファイルに入れておくだけ。
あとは以下のような感じでパスごとに環境を列挙してあげれば自動でclaudeやcodexの環境が切り替わるようになる。

```toml title="$XDG_CONFIG_HOME/agentenv/config.toml"
[path."$HOME/ghq/github.com/namachan10777"]
env = "default"
```
